- **연구 및 개발에 필요한 기술 스택 정리**
1. 프로젝트에서 자신이 담당한 기술의 정의
    - Selenium: 크롤링 데이터 수집
    - python : 파이썬을 활용한 데이터 전처리 진행
    - Word2Vec, CounterVertorizer: 모델링을 통한 기능 구현
    - MongoDB : 데이터 적재
    - Flask : flask를 활용하여 웹 서비스 기본 기능 구현.
2. 해당 기술(또는 연구방법)의 최근 장점과 단점
    - selenium
        - 장점
            - 크롤링을 자동화 할 수 있다.
            - 자바스크립트가 동적으로 만든 데이터를 크롤링 할 수 있다.
        - 단점
            - 브라우저를 직접 켜서 진행하기 때문에 자원을 많이 잡아 먹을 수 있다.
            - 사용자에게 제공되는 눈에 보이는 컨텐츠는 전부 크롤링이 가능하지만 컴퓨터 사양에 따라 느릴 수 있다.
    - Word2Vec
        - 장점
            - 단어를 다차원 벡터 공간에 표현하여 벡터간의 유사도를 계산할 수 있다.
            - 앞뒤 단어를 고려하여 임베딩 하기 때문에 단어의 문맥 상의 의미까지 정량화된 벡터로 표형 가능하다.
        - 단점
            - 사용자가 지정한 주변 단어의 개수에 대해서만 학습이 이루어지기 때문에 데이터 전체에 대한 정보를 담기 어렵다.
    - CounterVectorizer
        - 장점
            - 횟수를 사용해서 벡터를 만들기 때문에 직관적이고 간단해서 여러 상황에서 사용할 수 있다.
        - 단점
            - 횟수만을 특징으로 잡기 때문에 큰 의미가 없지만 자주 사용되는 단어들, 조사 또는 지시 대명사가 높은 값을 가지기 때문에 유의미하게 사용하기 어려울 수 있다.
    - Flask
        - 장점
            - 밑바닥부터 웹을 구성할 수 있다. 매우 가볍게 웹을 설계 가능하다.
        - 단점
            - 다른 프레임 워크에 비해 기능이 많이 없다.
    
3. 기간 내 구현 가능한 범위 설정
    - 데이터를 활용하여 웹에 테마 여행지 추천 기능 구현

# 4. **결론(고찰)**

1. CP1 프로젝트 전반에 대한 고찰 진행
    1. **연구 및 개발 통해 알게된 점**
        1. 여행객의 리뷰 데이터를 보며 리뷰에서는 평점에서는 안보이는 여행지의 느낌이나 여행객의 감성을 더욱 깊게 볼 수 있었다.
        2. 자연어 처리 과정에 대해서 좀 더 깊게 알 수 있었다. 다양한 서비스를 자연어 처리를 활용하여 개발할 수 있다는 것을 알았다.
    2. **프로젝트 결과를 사회 혹은 비즈니스에 어떠한 방식으로 활용해 볼 수 있는지 ( ※ 긍정적인 측면으로)**
        1. 테마를 활용한 여행객들이 원하는 여행을 구성할 수 있는 기회의 창을 마련할 수 있다고 생각된다. 
        2. 여행지만 아니라 교통 수단, 음식 등 다양한 카테고리를 테마로 분류하여 좀 더 여행객들에게 다채로운 여행을 제공할 수 있다.  
    3. **연구 및 개발을 진행하며 수행하지 못했던 부분 및 아쉬웠던 부분 ( ※ 이유를 포함하여 작성)**
        1. 음식점과 경로 안내 등 다양한 여행 추천 서비스를 생각했지만, 기술의 부족으로 인해 데이터를 알맞게 수집하는 것에 어려움을 겪어 시간의 부족을 경험하였다. 또한, 시간 부족으로 인해 배포 시도조차 못하고 끝난 것이 너무 아쉬웠다. 
        2. 데이터 수집 후 적재 과정까지는 잘했지만 적재한 데이터의 활용을 제대로 못한 것이 아쉽다. SQL과 NOSQL에 대한 이해를 더욱 깊게 해야겠다고 생각하였다.
    4. **향후 자신의 프로젝트 주제를 이어서 진행할 연구자 혹은 개발자를 위한 방향성 및 기술적 측면 제언** 
        1. 여행지 추천 서비스는 찾아보면 매우 많이 볼 수 있다. 하지만 거기에 있는 여행지, 음식점, 경로 추천은 매우 많은 기능이라고 생각된다. 하나의 서비스 구현에 집중한 뒤 다른 기능을 구현해 나가는 것이 좀 더 효율적인 접근법이라고 생각된다. 욕심이 나더라도 좀만 덜어내는 것이 좋다고 생각된다.
        2. 기술적으로는 api활용, json데이터에 대한 이해, DB에 대한 이해가 매우 많이 필요하다고 생각된다.
